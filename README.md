## 核心功能

1. 图文日记记录
   - 支持文本 + 多图（`png/jpg/jpeg/gif/webp/bmp`）。
   - 图片自动重命名并归档到本地 `image/` 目录。

2. 标签系统与多维筛选
   - 支持自定义标签。
   - 支持关键词搜索、按标签过滤、按日期筛选，并可组合使用。

3. 收藏与置顶
   - 重要日记可收藏（Favorite）。
   - 常用内容可置顶（Pinned）。

4. 删除保护（图片回收站）
   - 编辑/删除包含图片的日记时，旧图片先进入 `image/.trash/`，降低误删风险。

5. 导入导出与版本校验
   - 支持数据导出与导入。
   - 导出数据包含版本与校验信息，导入时会进行完整性检查。

6. 本地访问安全限制
   - 服务监听本地端口 `3765`，仅允许 Electron 客户端环境访问。

7. 存储路径自定义（新功能）
   - 支持在应用设置中自定义：
     - 数据库目录 `dbDir`
     - 图片根目录 `imageBaseDir`
   - 保存设置后会自动迁移 `timeline.db` 与 `image/` 数据。
   - 迁移失败会回滚，尽量保证数据一致性。
   - 设置会写入 `storage-settings.json`，下次启动自动生效。

## 存储结构说明

默认情况下，数据存放于：`%LOCALAPPDATA%\timeline`

主要文件/目录：

- `timeline.db`：日记正文、标签、时间、状态等元数据（SQLite）
- `image/`：日记图片
- `image/.trash/`：图片回收站
- `storage-settings.json`：自定义存储路径配置

## 使用说明（简版）

1. 安装并启动应用。
2. 在主界面写入文字、插入图片、设置标签/收藏/置顶。
3. 通过关键词、标签、日期快速检索历史记录。
4. 定期使用导出功能做备份。
5. 如需迁移目录，进入设置页修改存储路径并保存。

## 数据安全建议

- 不要手动删除 `timeline.db`、`image/`、`storage-settings.json`。
- 大版本升级、换机前先做一次导出备份。
- 导入属于覆盖写入，建议先备份当前数据再导入。
